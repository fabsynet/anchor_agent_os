   - Install: `cd apps/admin && pnpm add @tanstack/react-table`

   - Row click navigates to /agencies/[id]
   - Search input above table (debounced, filters name)

1. **Agency detail page** (/agencies/[id]/page.tsx): 'use client' component.
   - Use React.use(params) for Next.js 16 dynamic route params
   - Fetch agency detail: GET /admin/agencies/:id
   - Header: agency name, status badge, created date
   - Action buttons row: Suspend/Unsuspend, Edit Limits, Export Data
   - AgencyDetailTabs component below actions

2. **AgencyDetailTabs** (agency-detail-tabs.tsx): Tabbed view per locked decision (overview, users, policies summary, activity log).
   - **Overview tab**: Summary cards (user count, client count, policy count, document count, expense count). Agency info (name, slug, phone, province, limits). Created/updated dates.
   - **Users tab**: Table of agency users (name, email, role, status, created). Row actions: view user detail (navigate to /users page with filter). Show deactivated users with visual indicator.
   - **Policies Summary tab**: Policy counts by status (active, expired, pending_renewal, etc.) and by type (auto, home, life, etc.). Simple bar charts or stat cards. Fetches GET /admin/agencies/:id/policies-summary.
   - **Activity Log tab**: Paginated activity events for this agency. Fetches GET /admin/agencies/:id/activity. Shows event type, description, client name, timestamp. Paginated with load more.

3. **SuspendDialog** (suspend-dialog.tsx):
   - Uses ConfirmationDialog with variant='destructive'
   - Requires reason text (reasonRequired=true)
   - Calls POST /admin/agencies/:id/suspend with { reason }
   - On success, shows toast and refreshes page data
   - For unsuspend: same dialog but with variant='default' and optional reason

4. **LimitsDialog** (limits-dialog.tsx):
   - Modal with form: userCap (number input), storageCap (number input in MB)
   - Pre-populated with current values
   - Calls PATCH /admin/agencies/:id/limits
   - Validation: userCap 1-50, storageCap 100-10000

5. **ExportButton** (export-button.tsx):
   - Dropdown button with two options: Export as JSON, Export as CSV
   - Calls GET /admin/agencies/:id/export
   - JSON: downloads as .json file via Blob + URL.createObjectURL
   - CSV: converts to CSV via PapaParse unparse, downloads as .csv
   - Loading state during export
