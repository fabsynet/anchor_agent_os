1. **SuperAdminGuard** (super-admin.guard.ts):
   - Implements CanActivate
   - Injects PrismaService and ConfigService
   - Extracts Bearer token from Authorization header
   - Validates token via Supabase auth.getUser() (same pattern as JwtAuthGuard)
   - Looks up user.id in prisma.superAdmin where isActive = true
   - If not found, throw ForbiddenException('Not a super-admin')
   - Sets request.user = { id, email, firstName, lastName, isSuperAdmin: true }
   - CRITICAL: Does NOT set CLS tenant context (admin queries are cross-tenant)
   - Uses createClient from @supabase/supabase-js with anon key (same as JwtAuthGuard pattern)

2. **AuditController** (audit.controller.ts):
   - @Controller('admin/audit-logs')
   - @UseGuards(SuperAdminGuard) on the controller
   - Injects AuditService (already created in Plan 01, available via AuditModule)
   - GET / — list audit logs with query params (action, targetType, superAdminId, startDate, endDate, page, limit)
   - GET /stats — audit log statistics
  </action>

1. **AdminService** (admin.service.ts):
   - Injects PrismaService (use raw this.prisma, NOT tenantClient)
   - `getPlatformMetrics()`: Cross-tenant counts using Promise.all:
     - totalAgencies: prisma.tenant.count()
     - totalUsers: prisma.user.count()
     - totalPolicies: prisma.policy.count()
     - totalClients: prisma.client.count()
     - totalPremiumValue: prisma.policy.aggregate({ _sum: { premium: true } }) on active/pending_renewal policies
   - `getPlatformMetricsForRange(start: Date, end: Date)`: Same counts but filtered by createdAt within range
   - `getGrowthTimeSeries(startDate?: Date, endDate?: Date)`: Generate time series data between startDate and endDate. Default to last 12 months if no dates provided. For each month in the range, count new agencies/users/clients. Return array of { month: 'MMM yyyy', agencies, users, clients }. Use date-fns startOfMonth/endOfMonth/subMonths/format/eachMonthOfInterval.
   - `getEngagementMetrics()`:
     - activeAgencies (agencies with users who have updatedAt in last 30 days)
     - inactiveAgencies (agencies with no user activity in 30+ days)
   - `getHealthAlerts()`: Array of HealthAlert objects:
     - Email failure rate: count EmailLog where status contains 'fail' / total EmailLog in last 24h. Alert if > 10%
     - Inactive agencies: count tenants where all users have updatedAt > 30 days ago. Alert with count
     - Storage warnings: (placeholder — return empty for now since storage tracking isn't implemented per-tenant)

2. **AdminController** (admin.controller.ts):
   - @Controller('admin')
   - @UseGuards(SuperAdminGuard) on the controller
   - GET /admin/metrics — returns PlatformMetrics (optional query params: startDate, endDate for range filtering)
   - GET /admin/metrics/growth — accepts query params: startDate (ISO string), endDate (ISO string). Both optional — defaults to last 12 months. This enables Plan 05's time range selector to pass exact date ranges (7d, 30d, 90d, custom) without needing a months-to-date conversion.
   - GET /admin/metrics/engagement — returns engagement metrics
   - GET /admin/health — returns health alerts array

3. **AdminModule** (admin.module.ts):
   - Imports: ConfigModule, AuditModule (from ./audit/audit.module — created in Plan 01)
   - Providers: AdminService, SuperAdminGuard
   - Controllers: AdminController, AuditController
   - Exports: AuditModule (re-export so other modules importing AdminModule get AuditService)

4. **app.module.ts**: Add AdminModule to imports array.

IMPORTANT: All queries use raw this.prisma (NOT tenantClient). Admin services have no CLS tenant context.
