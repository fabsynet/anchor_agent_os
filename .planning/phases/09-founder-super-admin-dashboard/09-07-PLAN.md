  <action>
1. **ImpersonationProvider** (impersonation-provider.tsx): React context for impersonation state.
   - Context provides: isImpersonating, targetAgency, expiresAt, startImpersonation(userId), endImpersonation()
   - `startImpersonation(userId)`:
     a. Save current admin session access_token and refresh_token to a secure cookie (admin_session) with 30 min max-age
     b. Call POST /admin/impersonation/start with { targetUserId: userId }
     c. Receive { tokenHash, email, expiresAt, tenantId, tenantName }
     d. Call supabase.auth.verifyOtp({ type: 'email', token_hash: tokenHash, email }) to switch session to target user
     e. Store impersonation state in localStorage: { tenantName, expiresAt, targetUserId }
     f. Redirect to tenant app: window.location.href = `${process.env.NEXT_PUBLIC_TENANT_APP_URL || 'http://localhost:3000'}?impersonating=true`
   - `endImpersonation()`:
     a. Read admin_session cookie
     b. Call supabase.auth.setSession({ access_token, refresh_token }) from saved admin session
     c. Call POST /admin/impersonation/end with { targetUserId }
     d. Clear admin_session cookie and localStorage impersonation state
     e. Redirect back to admin app /users page
   - Auto-expiry: check expiresAt on mount and on interval (every 60s). If expired, auto-end impersonation.

2. **ImpersonationBanner** (update existing component from Plan 02):
   - Reads impersonation state from ImpersonationProvider context
   - When isImpersonating: Fixed amber/orange banner at top of page
   - Shows: "Acting as [Agency Name]" with agency icon (per locked decision)
   - "End Session" button that calls endImpersonation()
   - Shows remaining time (countdown from expiresAt)
   - Banner uses z-50 to stay above all content

3. **Update admin layout** ((admin)/layout.tsx):
   - Wrap children in ImpersonationProvider
   - Add ImpersonationBanner at the top of the layout (above sidebar/header)
   - Add NEXT_PUBLIC_TENANT_APP_URL to .env.local (default: http://localhost:3000)

Note: The impersonation flow redirects to the TENANT app (apps/web) where the super-admin will be logged in as the target user. When they return to the admin app, endImpersonation() restores their admin session. The tenant app does not need changes — it just sees a normal authenticated user.
