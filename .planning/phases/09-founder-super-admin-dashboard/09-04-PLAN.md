1. **AgenciesService** (agencies.service.ts):
   - Injects PrismaService (raw this.prisma, NOT tenantClient)
   - `getAgencyList(filters: AgencyListFilters)`: Paginated query on Tenant with optional search (name contains, case insensitive), isSuspended filter, sorting, include _count for users/clients/policies. Return { agencies, total, page, limit }.
   - `getAgencyDetail(tenantId: string)`: findUnique with include users (id, email, firstName, lastName, role, isActive, deactivatedAt, createdAt), _count for clients/policies/tasks/documents/expenses.
   - `getAgencyActivity(tenantId: string, page, limit)`: Query activityEvents where tenantId matches, ordered by createdAt DESC, paginated. Include client name.
   - `getAgencyPoliciesSummary(tenantId: string)`: Count policies by status and type for the agency.
   - `suspendAgency(tenantId: string, reason: string)`: Update tenant isSuspended=true, suspendedAt=now(). Returns updated tenant.
   - `unsuspendAgency(tenantId: string)`: Update tenant isSuspended=false, suspendedAt=null.
   - `updateAgencyLimits(tenantId: string, data: { userCap?, storageCap? })`: Update tenant limits.
   - `exportAgencyData(tenantId: string)`: Query all agency data (tenant, users, clients with policies, tasks, documents metadata, expenses). Return as structured JSON object (NOT files — frontend handles CSV/JSON formatting).

2. **AgenciesController** (agencies.controller.ts):
   - @Controller('admin/agencies')
   - @UseGuards(SuperAdminGuard)
   - GET / — list agencies (query: search, isSuspended, page, limit, sortBy, sortDir)
   - GET /:id — agency detail
   - GET /:id/activity — agency activity log (query: page, limit)
   - GET /:id/policies-summary — policies breakdown
   - POST /:id/suspend — suspend agency (body: { reason }). Inject AuditService (from AuditModule, available via AdminModule re-export) to log action.
   - POST /:id/unsuspend — unsuspend agency (body: { reason? }). Log via AuditService.
   - PATCH /:id/limits — update agency limits (body: { userCap?, storageCap? }). Log via AuditService.
   - GET /:id/export — export agency data as JSON

3. **JwtAuthGuard update** — Add tenant suspension check:
   After resolving tenantId (before setting CLS context), query `prisma.tenant.findUnique({ where: { id: tenantId }, select: { isSuspended: true } })`. If isSuspended is true, throw ForbiddenException('Your agency has been suspended. Contact support.'). This ensures suspended agency users cannot access the tenant API.

All controller mutating endpoints must call auditService.log() with the super-admin's ID (from request.user), action constant, targetType 'tenant', targetId, and relevant metadata.

1. **UsersService** (users.service.ts):
   - Injects PrismaService and Supabase admin client (createSupabaseAdmin pattern from supabase.config.ts)
   - `getUserList(params: { search?, tenantId?, isActive?, page, limit })`: Cross-tenant user list with search on email/firstName/lastName (case insensitive), optional tenantId filter. Include tenant { name }. Paginated, ordered by createdAt DESC.
   - `getUserDetail(userId: string)`: findUnique with tenant, include activity counts
   - `disableUser(userId: string)`: Update user isActive=false. Also call supabaseAdmin.auth.admin.updateUserById(userId, { ban_duration: 'none' }) — actually use `banned: true` to lock auth. Return updated user.
   - `enableUser(userId: string)`: Update user isActive=true, deactivatedAt=null. Call supabaseAdmin.auth.admin.updateUserById(userId, { banned: false }).
   - `deactivateUser(userId: string, reason?: string)`: Update user isActive=false, deactivatedAt=now(). Call supabaseAdmin.auth.admin.updateUserById(userId, { banned: true }). Soft delete — preserves data.
   - `changeUserRole(userId: string, role: UserRole)`: Update user role. If changing to admin, validate only one admin per tenant (or allow multiple — check existing pattern). Call supabaseAdmin.auth.admin.updateUserById to update user_metadata.user_role.
   - `triggerPasswordReset(userId: string)`: Look up user email, call supabaseAdmin.auth.admin.generateLink({ type: 'recovery', email }). This sends the reset email.

2. **UsersController** (users.controller.ts):
   - @Controller('admin/users')
   - @UseGuards(SuperAdminGuard)
   - GET / — list users (query: search, tenantId, isActive, page, limit)
   - GET /:id — user detail
   - POST /:id/disable — disable user. Log via AuditService.
   - POST /:id/enable — enable user. Log via AuditService.
   - POST /:id/deactivate — deactivate (soft delete). Log via AuditService.
   - PATCH /:id/role — change role (body: { role }). Log via AuditService.
   - POST /:id/password-reset — trigger password reset. Log via AuditService.

3. **ImpersonationService** (impersonation.service.ts):
   - Injects PrismaService, Supabase admin client, AuditService
   - `startImpersonation(superAdminId: string, targetUserId: string)`:
     - Look up target user (must be active, tenant must not be suspended)
     - Call supabaseAdmin.auth.admin.generateLink({ type: 'magiclink', email: targetUser.email })
     - Log impersonation.start via AuditService
     - Return { tokenHash: data.properties.hashed_token, email, expiresAt (30 min from now), tenantId, tenantName }
   - `endImpersonation(superAdminId: string, targetUserId: string)`:
     - Log impersonation.end via AuditService
     - Return success

4. **ImpersonationController** (impersonation.controller.ts):
   - @Controller('admin/impersonation')
   - @UseGuards(SuperAdminGuard)
   - POST /start — start impersonation (body: { targetUserId })
   - POST /end — end impersonation (body: { targetUserId })

   - Controllers: AgenciesController, UsersController, ImpersonationController (add to existing controllers)
