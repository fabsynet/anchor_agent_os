    **Birthday Methods:**

    1. `async sendBirthdayEmailsForAllTenants(): Promise<void>` -- Public method.
       - Early return if !this.zeptoApiKey (with warn log)
       - Fetch all tenants: `this.prisma.tenant.findMany({ select: { id: true, name: true } })`
       - For each tenant, try/catch `sendBirthdayEmailsForTenant(tenant.id, tenant.name)`

    2. `private async sendBirthdayEmailsForTenant(tenantId: string, tenantName: string): Promise<void>`
       - Check TenantEmailSettings: `this.prisma.tenantEmailSettings.findUnique({ where: { tenantId } })`
       - If settings exist and `birthdayEmailsEnabled === false`, return early (log skip)
       - If no settings, default to enabled (send emails)
       - Get today's month and day in Toronto timezone:
         ```typescript
         const torontoNow = new Date(new Date().toLocaleString('en-US', { timeZone: 'America/Toronto' }));
         const month = torontoNow.getMonth() + 1;
         const day = torontoNow.getDate();
         ```
       - Query clients with birthday today using `$queryRaw`:
         ```typescript
         const clients = await this.prisma.$queryRaw<Array<{ id: string; firstName: string; lastName: string; email: string }>>`
           SELECT id, first_name AS "firstName", last_name AS "lastName", email
           FROM clients
           WHERE tenant_id = ${tenantId}::uuid
             AND date_of_birth IS NOT NULL
             AND email IS NOT NULL
             AND email != ''
             AND EXTRACT(MONTH FROM date_of_birth) = ${month}
             AND EXTRACT(DAY FROM date_of_birth) = ${day}
         `;
         ```
       - For each client, check idempotency via EmailLog:
         ```typescript
         const alreadySent = await this.prisma.emailLog.findFirst({
           where: {
             tenantId,
             clientId: client.id,
             type: 'birthday_greeting',
             sentAt: { gte: startOfDay(torontoNow) },
           },
         });
         if (alreadySent) continue;
         ```
       - Render template: `const html = await render(BirthdayGreetingEmail({ clientName: client.firstName, agencyName: tenantName }));`
       - Send via `this.sendEmail({ to: client.email, toName: \`${client.firstName} ${client.lastName}\`, subject: \`Happy Birthday, ${client.firstName}!\`, html })`
       - Log to EmailLog regardless of success/failure:
         ```typescript
         await this.prisma.emailLog.create({
           data: {
             tenantId,
             clientId: client.id,
             recipientEmail: client.email,
             type: 'birthday_greeting',
             subject: `Happy Birthday, ${client.firstName}!`,
             status: result.success ? 'sent' : 'failed',
             errorMessage: result.error || null,
           },
         });
         ```

    **Renewal Reminder Methods:**

    3. `async sendRenewalReminderEmailsForAllTenants(): Promise<void>` -- Public method.
       - Same tenant-loop pattern as birthday. Early return if !this.zeptoApiKey.

    4. `private async sendRenewalReminderEmailsForTenant(tenantId: string, tenantName: string): Promise<void>`
       - Load TenantEmailSettings. Default all to true if no settings exist.
       - Build array of active intervals based on settings:
         ```typescript
         const intervals: number[] = [];
         if (settings?.renewalReminder60Days !== false) intervals.push(60);
         if (settings?.renewalReminder30Days !== false) intervals.push(30);
         if (settings?.renewalReminder7Days !== false) intervals.push(7);
         ```
       - For each interval, call `sendRenewalRemindersForInterval(tenantId, tenantName, daysBefore)`.

    5. `private async sendRenewalRemindersForInterval(tenantId: string, tenantName: string, daysBefore: number): Promise<void>`
       - Use Toronto timezone for "today" calculation (same as birthday).
       - Calculate target date: `const targetDate = startOfDay(addDays(torontoNow, daysBefore));`
       - Calculate next day: `const nextDay = startOfDay(addDays(targetDate, 1));`
       - Query policies expiring on targetDate:
         ```typescript
         const policies = await this.prisma.policy.findMany({
           where: {
             tenantId,
             status: { in: ['active', 'pending_renewal'] },
             endDate: { gte: targetDate, lt: nextDay },
           },
           include: {
             client: { select: { id: true, firstName: true, lastName: true, email: true } },
             createdBy: { select: { firstName: true, lastName: true, email: true } },
           },
         });
         ```
       - Filter out policies where client.email is null or empty.
       - For each policy, check idempotency (type = 'renewal_reminder', metadata contains policyId + daysBefore, sentAt >= startOfDay):
         ```typescript
         const alreadySent = await this.prisma.emailLog.findFirst({
           where: {
             tenantId,
             clientId: policy.client.id,
             type: 'renewal_reminder',
             sentAt: { gte: startOfDay(torontoNow) },
             metadata: { path: ['policyId'], equals: policy.id },
           },
         });
         if (alreadySent) continue;
         ```
         NOTE: If Prisma's Json path filtering doesn't work cleanly, fall back to checking tenantId + clientId + type + sentAt range, which is sufficient for daily idempotency.
       - Render template with policy details: policyType (use `policy.type` or `policy.customType`), policyNumber, format endDate, daysBefore, agent name/email from createdBy.
       - Send via `this.sendEmail()`.
       - Log to EmailLog with metadata: `{ policyId: policy.id, daysBefore }`.

    CRITICAL NOTES:
    - All methods use `this.prisma` directly (NOT tenant-scoped client). Cron has no CLS context.
    - All methods include manual `tenantId` in every query WHERE clause.
    - Use try/catch per-client/per-policy to prevent one failure from stopping others.
    - format() for date display: `format(policy.endDate, 'MMMM d, yyyy')` (date-fns already imported).

    ```typescript
      } catch (error) {
        this.logger.error(`Birthday email cron failed: ${error}`);
      }
    }
    ```
