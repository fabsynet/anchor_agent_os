    **1. Create CommunicationsService** (`communications.service.ts`):

    Inject PrismaService and NotificationsService. Import render from '@react-email/render' and the BulkAnnouncementEmail template.

    Methods:

    a) `async getEmailSettings(tenantId: string): Promise<TenantEmailSettings | default>`
       - Query: `this.prisma.tenantEmailSettings.findUnique({ where: { tenantId } })`
       - If null, return default values (all true). The frontend displays defaults when no record exists.

    b) `async updateEmailSettings(tenantId: string, dto: UpdateEmailSettingsDto): Promise<TenantEmailSettings>`
       - Use `this.prisma.tenantEmailSettings.upsert()`:
         - where: { tenantId }
         - create: { tenantId, ...dto }
         - update: { ...dto }
       - Return the upserted record.

    c) `async getEmailHistory(tenantId: string, query: EmailHistoryQueryDto): Promise<{ data: EmailLog[]; total: number; page: number; limit: number }>`
       - Build Prisma where clause: `{ tenantId }` + optional `type` filter + optional `clientId` filter.
       - Run count + findMany in parallel:
         ```typescript
         const where = { tenantId, ...(query.type && { type: query.type }), ...(query.clientId && { clientId: query.clientId }) };
         const [total, data] = await Promise.all([
           this.prisma.emailLog.count({ where }),
           this.prisma.emailLog.findMany({
             where,
             include: {
               client: { select: { id: true, firstName: true, lastName: true } },
               sentBy: { select: { id: true, firstName: true, lastName: true } },
             },
             orderBy: { sentAt: 'desc' },
             skip: ((query.page || 1) - 1) * (query.limit || 20),
             take: query.limit || 20,
           }),
         ]);
         ```
       - IMPORTANT: emailLog.count() is NOT overridden by tenant extension. Must include tenantId manually.
       - Return `{ data, total, page: query.page || 1, limit: query.limit || 20 }`.

    d) `async sendBulkEmail(tenantId: string, userId: string, dto: SendBulkEmailDto): Promise<{ sentCount: number; failedCount: number }>`
       - Fetch the tenant name: `this.prisma.tenant.findUnique({ where: { id: tenantId }, select: { name: true } })`
       - Fetch recipients based on filter:
         ```typescript
         const statusFilter = dto.recipientFilter === 'clients_only' ? 'client'
           : dto.recipientFilter === 'leads_only' ? 'lead' : undefined;
         const clients = await this.prisma.client.findMany({
           where: {
             tenantId,
             email: { not: null },
             ...(statusFilter && { status: statusFilter }),
           },
           select: { id: true, firstName: true, lastName: true, email: true },
         });
         ```
       - Filter out empty emails: `clients.filter(c => c.email && c.email.trim() !== '')`
       - If no recipients, return `{ sentCount: 0, failedCount: 0 }`.
       - Render template: `const html = await render(BulkAnnouncementEmail({ agencyName: tenant.name, subject: dto.subject, bodyHtml: dto.body }))`
       - Call `this.notificationsService.sendBatchEmail({ recipients: validClients.map(c => ({ email: c.email!, name: \`${c.firstName} ${c.lastName}\` })), subject: dto.subject, html })`
       - Log to EmailLog for EACH recipient (one row per client):
         ```typescript
         const logEntries = validClients.map(client => ({
           tenantId,
           clientId: client.id,
           recipientEmail: client.email!,
           type: 'bulk_announcement' as const,
           subject: dto.subject,
           status: result.success ? 'sent' : 'failed',
           errorMessage: result.error || null,
           sentById: userId,
           metadata: { recipientFilter: dto.recipientFilter || 'all' },
         }));
         await this.prisma.emailLog.createMany({ data: logEntries });
         ```
       - Return `{ sentCount: result.success ? validClients.length : 0, failedCount: result.success ? 0 : validClients.length }`.

    **2. Create CommunicationsController** (`communications.controller.ts`):

    Use `@UseGuards(JwtAuthGuard)` on the controller class. Import from guard file.
    Use `@Controller('api/communications')`.

    Endpoints (IMPORTANT: static routes before param routes):

    a) `GET /api/communications/settings` -- Get email settings for tenant
       - Extract user from `@Req()`: `req.user` has `tenant_id`.
       - Call `this.communicationsService.getEmailSettings(user.tenant_id)`
       - Return settings object.

    b) `PATCH /api/communications/settings` -- Update email settings (admin only)
       - Check `if (user.role !== 'admin') throw new ForbiddenException()`
       - Call `this.communicationsService.updateEmailSettings(user.tenant_id, dto)`

    c) `GET /api/communications/history` -- Get email history
       - `@Query()` with EmailHistoryQueryDto
       - Call `this.communicationsService.getEmailHistory(user.tenant_id, query)`

    d) `POST /api/communications/send` -- Send bulk email (admin only)
       - Check `if (user.role !== 'admin') throw new ForbiddenException()`
       - `@Body()` with SendBulkEmailDto
       - Call `this.communicationsService.sendBulkEmail(user.tenant_id, user.id, dto)`
       - Return 202 status code (use `@HttpCode(202)` decorator or return with appropriate status).
         Actually for simplicity, return 200 with the result object. The batch API is fast enough for <500 recipients.
